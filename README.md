# Stock Market Transaction Analytics Database

Relational database project for stock-market data, portfolio transactions, analytical SQL queries, and dashboard-ready reporting.

## Overview

This project designs a relational database for stock-market analysis and transaction management. The name makes the purpose explicit: stock data, transactions, analytics, and database design.

The work is structured around moving beyond spreadsheets into a normalized schema that can support real analytical queries and update flows.

## Project Objective

Stock-market records can include prices, trades, users, sectors, news, analyst ratings, and company events. Without a proper relational design, queries become fragile and transaction updates become difficult to control.

## Completed Scope

- Normalized stock-market schema
- Historical prices, user portfolios, transactions, and market information
- Analytical SQL query design
- Dashboard-ready reporting output

## Workflow

- Designed table relationships for stock and transaction entities
- Defined constraints to protect data integrity
- Built SQL queries for business and analytical questions
- Connected query output to reporting and visualization needs

## Deliverables

- Phase reports for a real-time stock market database
- Final presentation notes describing schema, SQL queries, transactions, and dashboard output
